# Artifact-Pyramid Phase Outputs

All SkillOpt phase outputs are stored in a single unified artifact-pyramid at the state directory root. No scattered directories.

## Unified Structure

```
<state_dir>/
├── 00-index.md                        ← amended per epoch: navigation to all epochs
├── 01-summary/findings.md             ← amended per epoch: run summary (final epoch, scores)
├── 02-analysis/
│   ├── epoch-trajectory.md            ← amended per epoch: pass rate trend table
│   └── epoch-<N>-overview.md          ← new per epoch: links to all phase dossiers
└── 03-dossiers/                       ← flat, no subdirectories
    ├── epoch-<N>-baseline.json
    ├── epoch-<N>-rollout-task-<X>.json
    ├── epoch-<N>-reflection.json
    ├── epoch-<N>-proposals.json
    ├── epoch-<N>-validation-edit-<X>.json
    ├── epoch-<N>-slowmeta.json
    └── ...
```

Naming convention: `epoch-<N>-<phase>[-<item>].json` — all files flat in `03-dossiers/`.

## Root Pyramid (Regenerated by `update_root_pyramid()`)

| File | Content | Amended? |
|------|---------|----------|
| `00-index.md` | Navigation links to all epochs, provenance | Yes — updated when any phase writes |
| `01-summary/findings.md` | YAML frontmatter: final epoch, skill name, target, pass rate trajectory | Yes — rewritten from board-metadata.json |
| `02-analysis/epoch-trajectory.md` | Table of pass rate, accepted/rejected per epoch | Yes — appended from pass_rate_history |
| `02-analysis/epoch-N-overview.md` | Per-epoch summary with SOURCES links to all 03-dossiers/ files | New file per epoch |

## Dossier Naming

| Phase | Filename pattern | Consumer |
|-------|-----------------|----------|
| Baseline | `epoch-N-baseline.json` | Validate phase (via `load_or_create_baseline()`) |
| Rollout | `epoch-N-rollout-task-<id>.json` | Reflect phase (reads from 03-dossiers/ first) |
| Reflection | `epoch-N-reflection.json` | Propose phase (reads from 03-dossiers/ first) |
| Proposals | `epoch-N-proposals.json` | Validate phase (reads from 03-dossiers/ first) |
| Validation | `epoch-N-validation-edit-<id>.json` | Merge phase (reads from 03-dossiers/ first) |
| Slow-meta | `epoch-N-slowmeta.json` | Human review only |

## Shared Module

`scripts/pyramid_utils.py` exports `update_root_pyramid(state_dir, epoch)` which scans `03-dossiers/`, discovers all epochs and phases, and rewrites the root pyramid files. Imported by all PYEOF blocks via `PYTHONPATH`.

## Backward Compatibility

Old-format directories (`baseline/epoch-N/`, `validation-results/`, `rollouts/`, `reflections/`, `proposals/`) are still checked by consumer read paths as fallbacks. New writes go only to `03-dossiers/`.

## Key Files

- `scripts/run-phase.sh` — All phase output generation
- `scripts/pyramid_utils.py` — Shared `update_root_pyramid()` function
- `scripts/archive-run.sh` — Generates root pyramid at archive time
- `references/artifact-formats.md` — JSON schemas for individual dossier content
