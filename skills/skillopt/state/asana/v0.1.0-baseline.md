# Asana CLI

A thin command-line wrapper around the Asana REST API (v1.0). Covers projects,
tasks, sections, users, teams, portfolios, goals, tags, custom fields, stories,
attachments, webhooks, workspaces, memberships, and status updates. Auth is
via Personal Access Token (PAT). No external SDK required — uses `requests`.

Does NOT cover OAuth flows, SCIM, audit log events, rule actions, or UI
Builder. For those, use the Asana MCP server or direct API calls.

## When to Use

- "list my projects"
- "show me tasks for project X"
- "create a task"
- "what teams am I on"
- "add a section to project Y"
- "who are the users in my workspace"
- "search for tasks matching Z"

## Prerequisites

1. **Personal Access Token** from https://app.asana.com/0/developer-console
2. Set `export ASANA_ACCESS_TOKEN="your_token_here"` in your shell profile
   or pass `--token TOKEN` on every invocation.

## How to Run

Invoke through the `terminal` tool:

```
terminal(command="asana projects list")
terminal(command="asana tasks list --project PROJECT_GID")
```

Aliased to `asana` via the script at `scripts/asana`. The script is
self-contained and shows help with no args:

```
terminal(command="asana --help")
```

## Procedure

### 1. Set up authentication
```
export ASANA_ACCESS_TOKEN="1/your_personal_access_token"
```

### 2. Basic workflow: list workspaces → projects → tasks
```
asana workspaces list
asana projects list --workspace <gid>
asana tasks list --project <gid>
```

### 3. Create a task with all common fields
```
asana tasks create \
  --name "Design new landing page" \
  --notes "Create mockups in Figma first" \
  --project PROJECT_GID \
  --assignee "me" \
  --due-on "2026-07-15" \
  --start-on "2026-07-01"
```

### 4. Move a task between sections
```
asana sections list PROJECT_GID
asana sections add-task SECTION_GID --task TASK_GID
```

### 5. Use raw JSON output for scripting
```
asana --json tasks list --project PROJECT_GID | jq '.data[] | {gid, name, due_on}'
```

### 6. Check for overdue tasks (filter client-side)
```
asana --json --opt-fields "name,due_on,completed" tasks list --project PROJECT_GID --completed-since "2020-01-01" | python3 -c "
import json,sys,datetime
data=json.load(sys.stdin)
today=datetime.date.today().isoformat()
for t in data.get('data',[]):
    due=t.get('due_on')
    if due and not t.get('completed') and due < today:
        days=(datetime.date.today()-datetime.date.fromisoformat(due)).days
        print(f'[{days}d] {t[\"name\"]} (due {due})')
"
```

To scan ALL projects for overdue tasks (batch pattern):
```python
import json, subprocess, datetime
# Get all projects
projs = json.loads(subprocess.run(['asana','--json','projects','list','--workspace','GID'], capture_output=True, text=True).stdout)['data']
# Collect overdue from each
for p in projs:
    result = subprocess.run(['asana','--json','--opt-fields','name,due_on,completed','tasks','list','--project',p['gid'],'--completed-since','2020-01-01'], capture_output=True, text=True)
    for t in json.loads(result.stdout).get('data',[]):
        due=t.get('due_on')
        if due and not t.get('completed') and due < datetime.date.today().isoformat():
            days=(datetime.date.today()-datetime.date.fromisoformat(due)).days
            print(f'[{days}d] {p[\"name\"]}: {t[\"name\"]} (due {due})')
```

## Pitfalls

- **Rate limits:** Asana uses cost-based rate limiting. If you get HTTP 429,
  wait for the `Retry-After` header duration before retrying.
- **Pagination:** The CLI returns paginated results. For >100 items, use
  `--limit 100 --offset <next_page_gid>`.
- **opt_fields is a global argument:** By default, Asana returns minimal fields (just `gid`, `name`, `resource_type`). To get richer data, add `--opt-fields "name,notes,assignee,due_on,completed,start_on"` **before the subcommand** — it's a global argument, not per-command. Correct: `asana --json --opt-fields "name,due_on,completed" tasks list ...`. Wrong: `asana tasks list --opt-fields "..." ...` (argparse ignores it and returns empty output). Do NOT use curl or direct API calls — the CLI tool is the interface.
- **Workspace required:** Many endpoints (create project, list users, list
  tags) require a `--workspace` parameter. Get workspace GIDs from
  `asana workspaces list`.
- **Project search is Premium:** `asana search tasks` requires an
  Enterprise or Business tier Asana account.
- **Section deletion:** Sections must be empty to delete. The last section
  of a project cannot be deleted.
- **Tasks max 30 dependencies:** Both `dependencies` and `dependents` lists
  are capped at 30 items per task.
- **Attachments limited to 100 MB:** File upload is multipart form-data.
  Non-ASCII filenames must be URL-encoded.

## Verification

Run a read-only command to confirm the tool works:

```
terminal(command="asana --json workspaces list")
```

Expect a JSON object with `{"data": [{"gid": "...", "name": "...", ...}]}`.
